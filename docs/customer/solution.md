# 卓鹰世纪费用管理解决方案

> 客户：武汉卓鹰世纪科技有限公司
> 版本：v0.1
> 日期：2026-04-11
> 状态：草案

---

## 1. 背景与目标

### 1.1 客户现状

卓鹰世纪已完成一期自研CRM建设，实现了"线索→客户→项目"的全流程管理。当前费用管理存在以下痛点：

- **报销分散**：报销流程在易快报运行，不在统一入口
- **流程碎片化**：宜搭中为每个项目单独设置了费用申请流程，数量多、维护成本高
- **统计困难**：费用数据分散在易快报和宜搭多个流程中，无法按项目维度统一归集统计
- **数据割裂**：费用数据与一期CRM项目数据未打通

### 1.2 建设目标

以**费用统计归集**为核心目标，在宜搭构建统一的费用管理流程，替代逐项目流程；将易快报报销流程迁移至宜搭；通过项目标签实现费用的自动归集和统计，为后续项目级利润核算奠定数据基础。

### 1.3 建设原则

- **统一入口**：所有费用相关操作收敛到宜搭一个平台
- **项目维度**：以项目为核心标签，贯穿申请→审批→支付→统计全链路
- **配置驱动**：通过审批规则配置替代多套独立流程
- **渐进交付**：分阶段实施，每阶段交付可见成果

## 3. 方案设计

### 3.1 统一费用管理流程

**目标**：替代当前每个项目独立的费用申请流程，构建 1-3 个统一流程。

#### 3.1.1 流程设计

```
费用申请人提交
    │
    ├─ 选择关联项目（从一期CRM项目列表中选择）
    ├─ 选择费用类型（差旅/招待/交通/办公/福利/管理/其他）
    ├─ 填写金额、事由、日期
    ├─ 上传发票/凭证附件
    │
    ▼
  审批流（按配置规则动态路由）
    │
    ├─ 规则1：按费用类型分流
    │   └─ 差旅费 → 部门负责人→财务总监
    │   └─ 招待费 → 部门负责人→分管领导→财务总监
    │   └─ 交通费 → 部门负责人
    │   └─ ...
    │
    ├─ 规则2：按金额阈值升级
    │   └─ ≤5000元：部门负责人审批
    │   └─ 5000~20000元：部门负责人→财务总监
    │   └─ >20000元：部门负责人→财务总监→李琼芳
    │
    ▼
  审批通过 → 进入待支付池
  审批驳回 → 退回申请人
```

#### 3.1.2 表单字段设计

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 关联项目 | 关联字段 | 是 | 从一期CRM项目表下拉选择 |
| 费用类型 | 下拉选择 | 是 | 差旅/招待/交通/办公/福利/管理/其他 |
| 申请金额 | 数字 | 是 | 元，支持两位小数 |
| 费用日期 | 日期 | 是 | 费用发生日期 |
| 费用事由 | 多行文本 | 是 | 简要说明费用用途 |
| 发票附件 | 附件上传 | 是 | 支持多张发票 |
| 其他说明 | 多行文本 | 否 | 补充信息 |

### 3.2 易快报报销流程迁移

**目标**：将易快报中全部报销流程迁移至宜搭，确保数据不断档。

#### 3.2.1 迁移策略

| 阶段 | 动作 | 产出 |
|------|------|------|
| 盘点 | 梳理易快报现有流程清单、字段、审批规则 | 迁移清单 |
| 映射 | 将易快报流程映射到统一费用流程 | 字段映射表 |
| 历史数据 | 导出易快报历史数据，清洗后导入宜搭 | 历史数据表 |
| 切换 | 停用易快报新流程，全部走宜搭 | 切换确认 |

#### 3.2.2 历史数据处理

- **方案A（推荐）**：导出易快报全量历史数据，按统一格式清洗后批量导入宜搭数据表，纳入项目统计
- **方案B（备选）**：保留易快报历史数据查询入口，新增数据走宜搭，报表层做双源聚合
- 选择依据：历史数据量级和字段匹配度

### 3.3 费用支付流程整合

**目标**：与统一费用申请流程打通，形成申请→审批→支付→归集闭环。

```
审批通过的费用申请
    │
    ▼
  进入待支付池
    │
    ▼
  财务人员选择待支付单据
    │
    ├─ 确认支付金额（可部分支付）
    ├─ 填写支付日期、支付方式（银行转账/现金/其他）
    ├─ 上传支付凭证
    │
    ▼
  支付完成
    │
    ▼
  自动更新项目费用汇总表
```

### 3.4 一期CRM项目数据联动

**目标**：确保费用管理中的项目数据与一期CRM保持一致。

#### 3.4.1 对接方案

| 对接项 | 方向 | 方式 | 频率 |
|--------|------|------|------|
| 项目列表 | CRM→宜搭 | API拉取/数据库同步 | 每日增量 |
| 项目状态 | CRM→宜搭 | API拉取/数据库同步 | 每日增量 |
| 项目变更 | CRM→宜搭 | Webhook推送（如支持） | 实时 |
| 费用数据 | 宜搭→CRM | API推送（可选） | 每日增量 |

#### 3.4.2 数据一致性保障

- 项目ID作为唯一关联键
- 定期（每日）对账：宜搭项目列表 vs CRM项目列表
- 异常告警：费用申请关联到已关闭/不存在的项目时拦截并告警

### 3.5 统计报表

**目标**：自动按项目维度汇总所有费用数据，支持多维度分析。

#### 3.5.1 报表类型

| 报表 | 维度 | 展示内容 |
|------|------|---------|
| 项目费用汇总 | 按项目 | 各项目费用总额、按类型分类、时间趋势 |
| 费用类型分析 | 按类型 | 各类型费用占比、趋势、异常波动 |
| 部门费用统计 | 按部门 | 各部门费用总额、人均费用、同比环比 |
| 时间维度汇总 | 按日/周/月 | 费用发生趋势、预算执行进度 |
| 审批效率分析 | 按审批人 | 平均审批时长、积压单据 |

#### 3.5.2 核心看板

```

├──────────────┬──────────────┬───────────────────┤

│ ¥ XXX,XXX   │ XX个          │ XX单              │
```

## 4. 实施计划

### 4.1 阶段划分

| 阶段 | 时间 | 交付内容 | 验收标准 |
|------|------|---------|---------|
| **Phase 0：现状盘点** | 1周 | 宜搭流程清单、易快报流程清单、一期CRM接口确认 | 完成三份清单确认 |
| **Phase 1：统一流程上线** | 2-3周 | 统一费用申请流程、审批规则配置、项目数据联动 | 新费用申请全部走统一流程 |
| **Phase 2：支付整合+报表** | 2-3周 | 费用支付流程整合、统计报表上线 | 可按项目维度查看费用汇总 |
| **Phase 3：迁移+收尾** | 1-2周 | 易快报历史数据导入、旧流程下线 | 历史数据完整纳入统计 |

**预计总周期：6-9周**

### 4.2 Phase 0 详细任务（立即执行）

| 任务 | 负责人 | 交付物 | 前置条件 |
|------|--------|--------|---------|
| 梳理宜搭费用相关流程清单 | 客户方IT/系统管理员 | 流程清单Excel | 无 |
| 梳理易快报报销流程清单 | 客户方财务 | 流程清单Excel | 无 |
| 调研一期CRM项目数据结构 | CRM负责人 | 数据字典+接口文档 | 无 |
| 确认宜搭审批流配置能力 | 实施方 | 能力评估报告 | 宜搭账号权限 |
| 确认易快报数据导出能力 | 实施方+客户财务 | 导出样例+字段说明 | 易快报账号权限 |

## 5. 成功标准

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| 费用流程数量减少 | 从N个逐项目流程整合为1-3个统一流程，减少80%+ | 迁移前后流程数量对比 |
| 费用统计效率提升 | 统计时间减少90%+ | 上线前后统计耗时对比 |
| 费用数据完整率 | 100%归集到对应项目 | 系统归集总额 vs 实际发生总额 |
| 费用归集准确率 | >99% | 抽样验证费用-项目关联正确性 |

## 6. 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 宜搭逐项目流程审批规则差异大，统一后覆盖不全 | 高 | 中 | Phase 0 全面梳理差异，通过配置化满足差异化需求 |
| 易快报历史数据字段不匹配 | 中 | 中 | 提前评估导出能力，制定数据清洗映射方案 |
| 一期CRM项目数据结构不规范 | 中 | 低 | 先调研数据质量，必要时做映射层 |
| 员工对流程变更不适应 | 低 | 中 | 上线前培训，统一后流程更简洁 |
